# 山东大学学生Minecraft社团服务器网络端口标准

> 生效日期：2026-06-29
>
> 维护团队：山东大学学生Minecraft社团技术部

## 一、网络拓扑结构说明

目前的网络架构采用“双校区内网穿透+单一公网入口”的模式，旨在解决多校区部署及NAT环境下的公网访问问题。具体方案为 [FRP范围端口穿透](./server_arch/env_cfg/frp/FRP-doc.md)

1. 公网入口节点（反向代理/转发层）
    - **公网服务器类型**：阿里云VPS(200mbps)
    - **公网IP/域名**：见技术部资料
    - **角色**：所有来自互联网的流量首先进入此节点，由该节点根据端口号将流量转发至对应的内网校区服务器。
    - **开放端口范围**：`25565`, `27000-28000`, `29000-30000` (TCP/UDP)
2. 内网后端节点（服务层）
    - 济南校区节点(`sducraft-mcsm-jn`)
        - 内网IP：`211.87.209.16`
        - 连接方式：通过内网穿透工具连接至外网节点。
        - 服务端口：映射至外网节点的 `25565` 及 `27000-28000` 段。
    - 青岛校区节点(`sducraft-mcsm-qd`)
        - 内网IP：`10.102.33.76`
        - 连接方式：通过内网穿透工具连接至外网节点。
        - 服务端口：映射至外网节点的 `29000-30000` 段。

## 二、端口映射与分配标准

为了便于管理和避免端口冲突，制定以下端口映射对照表及拓扑图。新服务器的开设必须遵循此标准。
| 服务器类型 | 服务区域 | 外网访问端口（入口） | 内网穿透映射（目标） | 备注 |
| --- | --- | --- | --- | --- |
| 周目生存服主服务器 | 济南校区 | 25565 | 211.87.209.16:25565 | 默认端口，无需指定 |
| 济南子服 | 济南校区 | 27000 - 27999 | 211.87.209.16:27000-27999 | 用于小游戏、测试服等 |
| 青岛子服 | 青岛校区 | 29000 - 29999 | 10.102.33.76:29000-29999 | 用于小游戏、测试服等 |
| 预留段 | - | 28000, 30000 | - | 作为上下限边界，建议不使用 |

## 三、后续使用说明与开设指南

当社团需要开设新的Minecraft服务器实例（如小游戏服、空岛服、模组服）时，参照文档的流程进行操作 [MCSM 服务端部署指南](./server_arch/env_cfg/mcsm/mcsm.md)：

**具体端口规范请严格按照以下流程操作：**

1. 确定服务器归属地
    - 济南校区：若服务器部署在 `211.87.209.16` 机器上，使用 `27xxx` 端口。
    - 青岛校区：若服务器部署在 `10.102.33.76` 机器上，使用 `29xxx` 端口。

2. 配置服务器实例 (Server Properties) - 
在启动新的Minecraft服务端（Spigot/Paper等）时，必须修改容器的端口映射参数。
    - 济南校区服务器：
        - `server-port=27xxx` (`27000-27999`，`25565`)
    - 青岛校区服务器：
        - `server-port=29xxx` (`29000-29999`)

3. 用户连接方式-
    向社团成员公布连接信息时，域名格式如下：
    - 校内玩家：`<内网IP>:<端口>`，如 `211.87.209.16:27001`
    - 公网玩家：`<域名>:<端口>`，如 `<domain>:27001`

## 四、注意事项

1. 端口唯一性：请务必确保在同一校区内，不同实例的端口号不重复。
2. 协议支持：外网节点已开放TCP和UDP协议，支持Minecraft原版及部分支持UDP的模组/插件。
3. 域名解析：目前 `<domain>` 已解析至 `<阿里云VPS公网IP>`，请勿随意更改`A`记录指向。
4. 故障排查：若出现无法连接，请依次检查：
```
本机服务是否启动 -> 本机防火墙 -> 内网穿透工具日志 -> 外网节点防火墙。
```
